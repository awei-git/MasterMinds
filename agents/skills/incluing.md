# Incluing（隐式世界构建）

## One-liner
通过角色行为、假设和随口提及来传递世界观设定，而不是停下来解释——SF写作的核心信息传递技术。

## The Problem
SF需要读者理解一个与现实不同的世界。自然的冲动是解释。解释打断叙事节奏，暴露作者在场，把读者从角色体验中拉出来。但不解释，读者看不懂。Incluing是解法：让设定信息搭载在角色的行为和认知上，而不是搭载在作者的声音上。

## The Pattern

```
1. EMBED  — 设定信息只能通过角色的行动、对话、感知进入文本。
             测试：删掉这句话，角色的行为是否仍然合理？
             如果是→信息没有真正嵌入，只是贴在表面。

2. IMPLY  — 用角色的反应传递规则，不用角色的嘴巴解释规则。
             反应暴露期望值，期望值暴露世界规则。
             测试：删掉角色的反应，读者是否仍能推导出规则？
             如果是→你直接陈述了规则，反应只是装饰。

3. LAYER  — 按推理依赖关系排序信息释放：先给理解动机所需的
             规则，再给理解冲突所需的规则，最后给理解主题所需
             的规则。
             测试：此刻读者是否已拥有推导这条规则的全部前置
             信息？如果没有→推迟。

4. TRUST  — 留出推理空白。读者自己推导出的设定比你告诉他们的
             更有说服力，因为推导过程制造投入感。
             测试：对于设定X，文中是否存在至少两条独立线索
             可以让读者三角定位出X？
             只有一条线索→谜语，不是空白。
             直接写出了X→答案，不是空白。
             正确状态：多条线索可推出X，X本身从未出现。
             例：设定X="这个社会以记忆交易为经济基础"。
             线索A：角色拒绝一笔交易，说"这段记忆是我女儿
             认得我的唯一原因"。线索B：街头广告写着"高价收
             购童年记忆，成色好价更高"。X从未被陈述，但两条
             线索交叉定位后读者自行得出。
```

## Techniques

**1. 假设即设定**
角色把世界规则当作理所当然。读者从"理所当然"中反推规则。
Bad:  "自从政府禁止了纸质书籍，人们只能在屏幕上阅读。"
Good: "她在黑市上摸到一本纸质书时，手指不知道该怎么翻页。"
（禁令、稀缺性、数字化常态——全部隐含在一个动作里。）

**2. 行话与自然命名**
角色使用他们世界的术语，不加解释。含义从使用场景中浮现。
Bad:  "他使用了一种叫做'跃迁'的超光速旅行技术。"
Good: "跃迁后他吐了三次，比上回好多了。"
（技术存在、有副作用、他经常使用——三个事实，零解释。）
关键：行话必须出现在有上下文线索的句子里。孤立的术语不是incluing，是噪音。

**3. 对比反应**
两个角色对同一事物的不同反应，揭示各自所处世界的不同规则。
Bad:  "这是一棵真正的树。在她的家乡，树木早在三十年前就灭绝了，但在他生活的地区，树木仍然随处可见。"
（作者跳出来为两个角色各写一段背景说明。）
Good: "她第一次看到真正的树时哭了。他不明白为什么——树到处都是。"
（两个世界的差异通过情感反差传递。读者从哭与困惑的对比中同时理解了两边——一边树木灭绝，一边树木寻常——不需要任何一方解释自己的背景。）

**4. 随口提及（passing reference）**
在叙述中提到设定元素，不解释，继续走。
"他绕过隔离区的边界——那些信标已经三年没人维护了——朝北走。"
（隔离区存在、正在衰败、至少三年了。叙事节奏不中断。）
使用位置：破折号插入语、从句、对话中的半句话。这些语法位置天然暗示"不重要到需要停下来"。

**5. 逆向info-dump：角色不解释已知事物**
Bad:  "你知道，跃迁引擎的工作原理是通过折叠空间的两个端点，" 林说，"所以我们实际上并没有超光速移动。"
（两个工程师互相解释他们每天都在用的基本原理。没有人在现实中这样说话。）
Good: "手册上说线圈能撑两百次跃迁，" 林说。"我们才跃迁了六十三次，衰减已经到红线了。" "那是因为手册假设你每次都做完整冷却周期。你做了吗？" "谁有那个时间？"
（引擎有线圈、线圈会衰减、有标准冷却流程、实际操作中没人遵守——四个设定事实通过一个关于当前任务的争论传递。）
诊断：如果对话中一个角色说了另一个角色已经知道的信息，你在写info-dump。

**6. 信息密度校准**
一个场景承载多少设定信息取决于场景的情感强度。高张力场景最多嵌入1个新设定元素——读者的注意力在情感上。低张力场景可以嵌入2-3个，因为读者有认知余量。
Bad:  追逐戏中：他跳上气垫车（新概念1），冲过阿尔忒弥斯区的关卡（新概念2），给老六发了一条脑链信息（新概念3）："黑莲的人找到我了（新概念4）。"
（读者在处理追逐的紧张感时，被迫同时消化四个全新概念。每个单独看都是合格的incluing，堆在一起全部失效——注意力被稀释，没有一个能被记住。）
Good:  追逐戏中只引入气垫车（通过驾驶动作自然嵌入）。阿尔忒弥斯区、脑链、黑莲分散到前后的低张力场景中——旅途闲聊提及阿尔忒弥斯区，等待时用脑链联系朋友，日常场景中有人提起黑莲的传闻。

## Trigger Conditions（逐句自检清单）

每条触发器 = 页面上你会看到的具体文字特征 + 诊断测试 + 修复路径。

| # | 你在页面上看到了什么 | 诊断测试 | 修复 |
|---|---|---|---|
| T1 | 一段文字里没有任何角色（无人说、做、看、想） | 把这段文字遮住，问：哪个角色的行为因此变得不可理解？如果答案是"没有"→这段信息没有叙事载体 | 用**技术#1假设即设定**重写：找一个角色，让这条信息成为他行为的前提而非旁白的注释 |
| T2 | 对话中角色A对角色B说了一个B已经知道的事实 | 删掉这句对话。B的下一句回复是否仍然成立？如果是→这句话是写给读者的，不是写给B的 | 用**技术#5逆向info-dump**重构：制造真实信息落差，或让设定搭载在任务争论上 |
| T3 | 一个新术语/概念第一次出现，后面紧跟一段解释（超过一个从句） | 遮住解释部分，只看术语出现的那个句子。句子本身是否提供了至少一条关于术语含义的上下文线索？如果没有→术语是孤立的噪音；如果有→解释多余 | 线索不足：在术语出现的句子里添加行为/感官上下文（**技术#2**）。解释多余：删掉解释，用**技术#4随口提及**在后续场景补第二条线索 |
| T4 | 出现"所谓的""也就是说""简单来说""——一种……的技术"等元解释标记 | 这些短语的功能是作者在对读者说"让我帮你翻译"。在角色的世界里，没有人需要翻译自己的日常用语 | 删掉元解释标记，确保术语所在的句子满足**技术#2**的自释条件（使用场景提供线索） |
| T5 | 两个角色的对话读起来像教科书问答（一问一答，每个回答都是完整信息块） | 读出声。真实对话的特征：打断、跑题、省略共享知识、情绪干扰信息传递。你的对话有几个这样的特征？如果 < 2 →对话是伪装的说明文 | 用**技术#3对比反应**或**技术#5**重写：给两个角色不同的立场，让设定信息作为争论弹药出现，而非教学内容 |
| T6 | 一个场景里你引入了3个以上全新设定概念 | 列出这个场景中读者第一次遇到的概念。数量超过该场景的张力预算（高张力≤1，低张力≤3）？ | 用**技术#6信息密度校准**拆分：保留与当前场景情感核心直接相关的概念，其余推迟到后续场景 |

## Pitfalls & Repairs

**1. 过度隐晦 → 锚点技术**
症状：Beta读者说"我不知道发生了什么"而不是"我想知道接下来会怎样"。前者是困惑，后者是好奇——区别在于你是否给了锚点。
修复：在前三章，给每个核心世界规则至少一个"高信噪比时刻"——一个场景中，角色的行为几乎只能用这个规则来解释，推理路径只有一条。不需要解释规则本身，但因果链必须无歧义。
例：如果你的世界里记忆可以交易——不要在第一章随口提到"记忆市场"然后跳过。给一个角色因为卖掉某段记忆而认不出自己的孩子。读者无法误读这个因果链。

**2. "你知道的，Bob"综合征 → 视角落差**
症状：两个都知道某事的角色互相解释。（参见触发器T2和技术#5。）
核心修复不是找一个"局外人"——这是俗套。而是制造**视角落差**：两个都知道跃迁技术的人，工程师关心怎么修，乘客关心会不会死。同一事实的不同关切面自然暴露不同层面的信息，无需任何一方扮演老师。

**3. 前后矛盾 → 设定圣经**
症状：第三章暗示的规则和第七章暗示的规则互相矛盾。Incluing分散在全文中，比info-dump更难自查一致性。
修复：维护一份独立的设定文档（不是给读者看的），列出每条世界规则及其在文中的所有暗示点。每次新增一个incluing时刻，回查这份文档。成本不高，但能防止最致命的错误——读者可以接受不完全理解，不能接受理解后发现你自相矛盾。

**4. 节奏坍塌 → 信息预算**
症状：单个场景中每条incluing单独看都合格，但堆在一起读者消化不了。（参见触发器T6和技术#6。）
修复公式：场景信息预算 = 场景张力级别决定的上限（高≤1，低≤3）。超出预算的概念不是删掉，是推迟——incluing的核心优势就是可以分散到全文。
