# 修稿反思（Feedback Loop）

每次修稿循环结束后，铁面（editor）必须输出一份简短的反思，供下一轮写作参考。

---

## 反思格式

```
## 本轮反思

### 反复出现的问题
- [列出本轮修稿中反复被指出的问题，1-3条]

### 有效的改进
- [列出修改后确实变好的地方，1-3条]

### 下次写作注意
- [给妙笔的具体提醒，1-3条。不是泛泛的"注意节奏"，而是"对话之间的动作描写不要超过一句"]
```

---

## 规则

1. **只记模式，不记个例**：反思要提炼出规律，不是重复某一处的意见
2. **具体到可执行**："对话中避免每句都带动作tag"比"注意对话自然度"有用一百倍
3. **不超过150字**：反思不是报告，是备忘录
4. **积累性**：每轮反思会被保存，妙笔下次写作时会看到之前的所有反思。所以不要重复已经记过的

---

## 使用方式

- **妙笔写新内容前**：读项目已有的反思列表，避免重复犯同样的错
- **铁面审稿时**：对照反思列表检查，看旧问题是否复发
- **创作者**：可以在反思中加入自己的观察和偏好
